# Fix Pre-Push Hook Blocking on Turbo Typecheck

## Root Cause

The `mergebranch` branch has modified multiple files, and **13 files** (including `package.json` and `packages/kilo-vscode/package.json`) have unintentionally acquired a **UTF-8 BOM** (`\uFEFF`) at the start of the file.

Turbo's JSON parser does not handle BOMs. When turbo cannot parse the root `package.json`, it cannot detect the `workspaces` field, defaults to single-package mode, and then rejects package-specific tasks like `@kilocode/cli#build` in `turbo.json`. This causes `bun turbo typecheck` to fail, which blocks the pre-push hook.

The bun version check in `.husky/pre-push` is NOT blocking — `bun@1.3.14` satisfies `^1.3.13`, so the `semver.satisfies` check passes. The warning is just `console.warn` and does not block. The only blocking issue is the turbo typecheck failure.

## Plan

### 1. Remove BOM from all affected files

13 files have BOMs. The critical ones for turbo workspace detection are:
- `package.json` (root workspace config)
- `packages/kilo-vscode/package.json` (workspace package config)

But all 13 should be cleaned to prevent future issues. Files with BOM:
- `.env.example`
- `AGENTS.md`
- `CODE_OF_CONDUCT.md`
- `COMMANDS.md`
- `CONTRIBUTING.md`
- `LICENSE`
- `PRIVACY.md`
- `README.md`
- `RELEASING.md`
- `SECURITY.md`
- `TESTING.md`
- `package.json`
- `packages/kilo-vscode/package.json`

Clean each file by reading, stripping BOM if present, and rewriting.

### 2. Verify the fix

Run `bun turbo typecheck` (or equivalently `bun typecheck`) to confirm it no longer fails with the "single-package repositories" error.

### 3. The bun version warning behavior already matches requirements

The pre-push hook already:
- Warns when bun version differs (non-blocking `console.warn` on line 17)
- Does NOT block on version mismatch (`semver.satisfies` check allows `^1.3.13` range, which matches 1.3.14)
- No changes needed to the hook
